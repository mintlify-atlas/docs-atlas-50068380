---
title: "REST API endpoints for customer orders"
description: "GET, POST, PUT, DELETE /orders — manage customer sales orders, add product lines, validate, and create shipments via the Dolibarr REST API."
---

Customer orders in Dolibarr capture confirmed commitments from your customers, forming the basis for stock reservation, warehouse picking, and delivery scheduling. An order progresses from **draft** (`statut = 0`) through **validated** (`statut = 1`) to **shipped** (`statut = 2`), and can be **cancelled** (`statut = 3`) at any point before fulfilment. Validating an order triggers stock reservation and makes the order available for shipment creation.

**Base path:** `/api/index.php/orders`

All requests require a valid API token passed in the `DOLAPIKEY` header.

## Create an order

Both `socid` and `date` are mandatory.

<ParamField body="socid" type="number" required>
  ID of the third party (customer) placing the order.
</ParamField>

<ParamField body="date" type="number" required>
  Order date as a Unix timestamp.
</ParamField>

<ParamField body="date_livraison" type="number">
  Expected delivery date as a Unix timestamp.
</ParamField>

<ParamField body="ref_client" type="string">
  Customer's own purchase order reference.
</ParamField>

<ParamField body="lines" type="object[]">
  Array of order line objects.

  <Expandable title="line properties">
    <ParamField body="fk_product" type="number">
      ID of the product or service.
    </ParamField>
    <ParamField body="product_label" type="string">
      Line description (used for free-text lines).
    </ParamField>
    <ParamField body="qty" type="number" required>
      Quantity ordered.
    </ParamField>
    <ParamField body="subprice" type="number" required>
      Unit price excluding tax.
    </ParamField>
    <ParamField body="tva_tx" type="number">
      VAT rate as a percentage.
    </ParamField>
  </Expandable>
</ParamField>

```bash Create a sales order
curl --request POST \
  --url 'https://your-dolibarr.example.com/api/index.php/orders' \
  --header 'DOLAPIKEY: your_api_key' \
  --header 'Content-Type: application/json' \
  --data '{
    "socid": 42,
    "date": 1745280000,
    "ref_client": "PO-2024-0099",
    "lines": [
      {
        "fk_product": 7,
        "qty": 10,
        "subprice": 45.00,
        "tva_tx": 20
      }
    ]
  }'
```

**Response:** Returns the numeric ID of the created order.

## Validate an order

```
POST /orders/{id}/validate
```text

Validates a draft order, triggering stock reservation and assigning the final order reference.

<ParamField path="id" type="number" required>
  ID of the draft order to validate.
</ParamField>

<ParamField body="idwarehouse" type="number">
  Warehouse ID for stock reservation.
</ParamField>

```bash
curl --request POST \
  --url 'https://your-dolibarr.example.com/api/index.php/orders/23/validate' \
  --header 'DOLAPIKEY: your_api_key' \
  --header 'Content-Type: application/json' \
  --data '{
    "idwarehouse": 1
  }'
```
